# Definition of Done

## Feature Completion Criteria

A feature is **DONE** when:

### Code
- [ ] Code is written and committed
- [ ] No console errors or warnings
- [ ] Handles edge cases gracefully
- [ ] Environment variables documented

### Testing
- [ ] Manual testing passed
- [ ] Happy path works end-to-end
- [ ] Error states handled

### Integration
- [ ] Works with watsonx Orchestrate
- [ ] API responses are correct
- [ ] UI displays results correctly

### Demo Ready
- [ ] Feature is demonstrable
- [ ] No hardcoded secrets
- [ ] Can explain in 30 seconds

---

## MVP Definition of Done

The MVP is **DONE** when:

1. User can submit an incident description
2. Agent classifies incident (severity + category)
3. Agent recommends owner assignment
4. Agent suggests next steps
5. Results display in UI
6. Demo script can be executed without errors

---

## Demo Checklist

- [ ] Fresh environment (clear cache/state)
- [ ] All API keys configured
- [ ] Demo data prepared
- [ ] Backup plan if live demo fails
- [ ] Timer set for presentation slot
